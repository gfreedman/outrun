# src/ — Source Files

All game logic lives here. TypeScript, compiled by esbuild into `dist/script.js`.

---

## Entry point

```
main.ts
  └── Creates the canvas, starts Game
```

`main.ts` is the only file that touches the DOM. It sizes the canvas to the
window, constructs `Game`, and calls `game.start()`. Everything else is
initiated from there.

---

## Module map

```
main.ts
  └── game.ts                  ← the game loop and phase state machine
        ├── intro-controller.ts  ← title screen / menu UI (extracted from game.ts)
        ├── road.ts              ← track definition and segment lookup
        │     └── road-data.ts   ← pre-generated segment array (build artifact)
        ├── renderer.ts          ← two-pass canvas renderer
        │     ├── renderer-hud.ts      ← speed, tach, timer overlays
        │     ├── renderer-menu.ts     ← title screen drawing
        │     └── renderer-screens.ts  ← goal/countdown/time-up overlays
        ├── physics.ts           ← pure physics functions (tick, collision response)
        ├── collision.ts         ← roadside obstacle detection
        ├── traffic.ts           ← AI car pool, movement, player/traffic collision
        ├── audio.ts             ← Web Audio engine (music, engine, SFX)
        ├── input.ts             ← keyboard state tracker
        ├── preloader.ts         ← sprite sheet image loading
        ├── sprites.ts           ← sprite sheet coordinates, TRAFFIC_CAR_SPECS
        ├── ui.ts                ← Button class (hover/click state)
        ├── types.ts             ← shared TypeScript interfaces
        └── constants.ts         ← every tuning value in one place
```

## File-by-file

### `constants.ts`
**Every tuning value in one place.**
Physics constants (max speed, steering, centrifugal, drift), camera geometry
(height, depth, road width, segment length), color palette (hex strings), draw
distance, sprite hitbox radii, collision window, traffic parameters.

If you want to change how something *feels*, start here.

---

### `types.ts`
**Shared TypeScript interfaces and enums.**
`RoadSegment`, `ProjectedPoint`, `SpriteInstance`, `SpriteFamily`, `GameMode`,
`CollisionClass`, `GameSettings`, `TrafficType`.

No logic — just type definitions used across multiple modules.

---

### `main.ts`
**Entry point. Canvas init and game start.**
Sizes the canvas to `window.innerWidth × window.innerHeight`. Constructs `Game`
and calls `game.start()`. Also handles `window.resize`.

---

### `game.ts`
**The game loop and phase state machine.**
Owns: `requestAnimationFrame` loop, delta-time accumulation, mouse event
listeners, canvas reference, all subsystem instances (renderer, audio, road,
physics, traffic, collision).

Four game phases, each with its own `tick` method:
- `INTRO` — title screen (delegates to `IntroController.tick()`)
- `COUNTDOWN` — 3-2-1-GO! sequence
- `PLAYING` — active race
- `GOAL` / `TIME_UP` — post-race result screen

On phase transition (e.g. START → COUNTDOWN), `beginRace()` initialises
the race state from `intro.settings`.

---

### `intro-controller.ts`
**Title screen / menu UI state machine. Extracted from game.ts.**
Owns: menu item focus, sub-menu open state, difficulty picker selection, sound
toggle, pulse clock, hero image loading, all nine menu buttons, settings load/save.

Separation contract: Game owns mouse event listeners (shared with in-game phases)
and passes mouse coordinates as parameters to `IntroController.tick()`. The only
cross-boundary call is `onStartRace()` — a callback injected at construction that
fires when the user confirms START.

Settings are persisted to `localStorage` and read back on startup.

---

### `road.ts`
**Track definition and segment lookup.**
`Road` builds the track from high-level primitives:
- `addStraight(length)` — flat straight section
- `addCurve(length, curve, hill)` — curved + optionally hilly section
- `addHill(length, hill)` — hill without curve
- `addSCurves()` — pre-composed S-curve pattern

Two courses: `'default'` (EASY/MEDIUM) and `'hard'`. Sprites (palms, shrubs,
cactus, billboards, houses, signs) are placed during construction.

`findSegment(worldZ)` returns the `RoadSegment` at a given world Z position
using integer division with wrapping.

---

### `road-data.ts`
**Pre-generated segment array. Build artifact — do not edit.**
Generated by `scripts/generate-road.ts` at build time. Contains
`ROAD_DATA` and `ROAD_DATA_HARD` as serialized segment arrays.
`game.ts` calls `Road.fromData(ROAD_DATA)` to skip the 50–200 ms construction
cost at runtime.

Regenerate with: `npm run prebuild`

---

### `renderer.ts`
**Two-pass canvas renderer.**

The rendering pipeline runs every frame:

**Pass 1 — Project (front to back)**
Walk segments from the player's position forward. For each visible segment,
compute screen coordinates (`sx1, sy1, sw1, sc1, sx2, sy2, sw2, sc2`) and write
into a pre-allocated `projPool` slot. `maxy` occlusion skips segments hidden
behind hill crests.

**Pass 2 — Road surface (back to front)**
Draw road trapezoids using `Math.round(sy)` to eliminate sub-pixel canvas alpha
bleed. Batched by color run to eliminate hairline seams.

**Pass 3 — Sprites (back to front)**
For each projPool slot, draw billboards, plants, and buildings scaled by `sc1`.

**Pass 4 — Traffic cars (back to front)**
For each traffic car, interpolate screen position *within* its segment using
`t = worldZ % SEGMENT_LENGTH / SEGMENT_LENGTH` to prevent vibration at segment
boundaries.

**Pass 5 — Player car + HUD**
Draw the Ferrari sprite at screen center, scaled to its world height. Then
overlay the HUD.

Exported functions (`drawRoadSurface`, `drawRumble`, `drawLaneDashes`,
`drawEdgeMarks`, `drawGrass`, `addTrap`) are unit-tested in `tests/renderer.test.ts`.

---

### `renderer-hud.ts`
**Speed, tachometer, and timer overlays.**
Draws the bottom-left HUD: 7-segment speed digits, 3-row LED tach bar,
lap timer, distance remaining. OutRun 1986 arcade palette.

---

### `renderer-menu.ts`
**Title screen drawing.**
Draws the hero image background, main menu items (MODE/SETTINGS/START),
mode picker sub-menu cards (EASY/MEDIUM/HARD), and settings panel.

---

### `renderer-screens.ts`
**Goal, countdown, and time-up screen overlays.**
Full-screen overlays drawn on top of the road during COUNTDOWN, GOAL,
and TIME_UP phases.

---

### `physics.ts`
**Pure physics functions, unit-testable.**
`advancePhysics(state, input, cfg, dt)` — single-frame tick. Mutates
`PhysicsState` in place: throttle, braking, coasting, steering, centrifugal drift,
off-road friction, grind deceleration, `playerZ` advance.

`applyCollisionResponse(state, hit)` — applies the result of a collision hit
descriptor to the physics state (speed reduction, shake, grind timer, etc.).

Both functions are pure (no global state, no DOM). Fully tested in
`tests/physics.test.ts`.

---

### `collision.ts`
**Roadside obstacle collision detection.**
`checkCollisions(playerX, segments, count, playerSegIdx)` — scans
`COLLISION_WINDOW = [-1, 0, 1, 2]` segments around the player and returns the
worst hit (or null).

`checkSegmentCollision(playerX, seg)` — checks one segment. Returns the worst
`CollisionHit` (ghost → glance → smack → crunch) and a `nearMiss` if the player
passed close but not close enough to collide.

On-road immunity: no collision when `|playerX| < COLLISION_MIN_OFFSET`.
Ghost families (shrub, sign) are always ignored.

---

### `traffic.ts`
**AI car pool, movement, and player/traffic collision.**
`initTraffic(segCount)` — allocates `TRAFFIC_COUNT` cars at random positions.

`updateTraffic(cars, playerZ, segCount, dt)` — advances each car's `worldZ`
by `speed × dt`. Cars that fall too far behind the player are recycled to
`DRAW_DISTANCE ± 5` segments ahead.

`checkTrafficCollision(playerX, playerZ, playerSpeed, cars, segCount)` — checks
depth window and lateral overlap. Returns a hit descriptor with `bumpDir` and
`closingSpeed`, or null.

---

### `audio.ts`
**Web Audio engine.**
Engine growl (two detuned oscillators tracking RPM), off-road screech, OutRun
BGM synthesis, finish fanfare. All audio is constructed procedurally using the
Web Audio API — no audio files.

---

### `input.ts`
**Keyboard state tracker.**
`InputManager` tracks which keys are currently held (`isDown`) and which were
pressed this frame (`wasPressed`). `wasPressed` flags are cleared each frame
after being consumed.

---

### `preloader.ts`
**Sprite sheet image loading.**
Loads all PNG sprite sheets asynchronously. Returns a promise that resolves
when all images are ready. A load failure is logged but non-fatal — the
renderer skips missing sheets.

---

### `sprites.ts`
**Sprite sheet coordinates and metadata.**
`SPRITE_INFO` — pixel coordinates of every sprite in every sheet.
`TRAFFIC_CAR_SPECS` — per-type dimensions and asset paths for all four
traffic car types. Used by the renderer and the traffic unit tests.

---

### `ui.ts`
**Button hover/click state.**
`Button` tracks whether the mouse is hovering and whether a click landed on it.
`btn.tick(mouseX, mouseY, clickX, clickY, mouseClick)` updates state each frame.
`anyHovered(...buttons)` returns true if any button is currently hovered
(used to set `cursor: pointer` on the canvas).

---

## Key architectural decisions

**Pre-allocated projPool** — `renderer.ts` allocates `DRAW_DISTANCE` pool slots
once at construction and reuses them every frame. No per-frame object allocation,
no GC pressure on the render path.

**Physics extraction** — `physics.ts` contains only pure functions. This makes
the physics unit-testable without a game loop or DOM, and ensures all frame-rate
dependent quantities use `Math.exp(-k*dt)` decay instead of `1 - k*dt`
approximations.

**IntroController extraction** — the 300-line menu state machine is separated
from `game.ts` so the physics and race logic aren't tangled with button hover
state and localStorage.

**Build-time road generation** — placing ~3000 sprites on the road takes 50–200 ms
synchronously. Done once by `scripts/generate-road.ts` at build time; runtime
loads the pre-computed array.

**road-data.ts is checked in** — the generated file is committed so CI doesn't
need `tsx` to run tests, and `git diff` shows exactly what changed when the
course layout is modified.
